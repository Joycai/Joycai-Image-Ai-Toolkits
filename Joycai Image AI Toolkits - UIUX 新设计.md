# Joycai Image AI Toolkits - UI/UX 设计规范

## 1. 全局布局架构 (Desktop 基准)

在桌面端，应用采用经典的**“三栏一底 + 独立导航”**的生产力工具布局（类似于 VS Code 或主流 3D 软件）。

- **最左侧 - App 全局导航栏 (Navigation Rail)**：极窄的一列，仅包含图标。用于切换 App 的顶级功能域（例如：工作台 Workbench、设置 Settings、模型管理 Models、系统监控等）。
- **左侧 - 可展开面板 (Left Panel)**：固定宽度（可拖拽调节），展示当前功能域下的资源列表或树状结构。包含折叠/展开按钮。
- **中央 - 主工作区 (Center Workspace)**：占据屏幕最大面积。包含：
  - **顶部工具栏 (Top Bar)**：分为两层。上层/左侧为当前子模块的 Tab 切换（图片处理、比较器等）；右侧为该模式下的全局操作（如：导入图片、清空视图）。
  - **内容展示区 (Main Content)**：画廊、对比视图或画布。
- **右侧 - 可展开面板 (Right Panel)**：固定宽度（可拖拽调节），充当“属性检查器 (Inspector)”。内容严格随中央工作区的状态动态变化。
- **底部 - 日志控制台 (Log Console)**：默认收起为一条带有状态指示灯（🟢 Running, 🔴 Error）的窄条。点击后向上展开，显示详细执行日志。

## 2. 核心工作台 (Workbench) 子模块设计

在“工作台”这个全局顶级域下，通过顶部 Tab 切换以下四个子模块。面板的内容会根据所选 Tab 自动切换。

### 2.1 图片处理 (Image Processing)

这是最高频的默认模式，用于看图和发送 AI 处理指令。

- **顶部工具栏**：网格缩放滑块、全选/取消全选、刷新。
- **左侧面板**：树状目录树 (Directory Tree)。
  - 包含常规本地文件夹。
  - **固定节点**：“临时工作区 (Temp)”和“输出结果 (Output)”，直接像文件夹一样点击切换，不再需要单独的 Tab。
- **中央工作区**：图片画廊 (Grid View)。卡片上提供显式的悬浮按钮（✨优化提示词、⚖️对比、🖌️遮罩）。
- **右侧面板**：控制区。包含 Model Selection（模型选择下拉框）和 Prompt（提示词输入框 + 提交按钮）。

### 2.2 比较器 (Comparator)

用于精细对比两张或多张图片的差异。

- **顶部工具栏**：视图切换（左右并排 Sync、覆盖滑动 Swap）、清空对比器。
- **左侧面板**：**自动隐藏**，释放屏幕空间。
- **中央工作区**：大面积对比视图。若缺少图片，显示巨大“+”号提示拖入或从图库发送。
- **右侧面板**：元数据检查器 (Metadata Inspector)。左右分栏或上下对比显示这两张图的生成参数（Prompt、Seed、LoRA 权重等）。

### 2.3 遮罩编辑器 (Mask Editor)

沉浸式的图片涂抹工作流。

- **顶部工具栏**：画笔大小、橡皮擦、撤销/重做。最右侧为高亮的 **“保存并返回”** 与 **“取消”** 按钮。
- **左侧面板**：**自动隐藏**。
- **中央工作区**：单张图片的完整画布，支持滚轮缩放和拖拽平移。
- **右侧面板**：**自动隐藏**。（如果未来有复杂的遮罩反选、羽化等高级参数，可在此处展出，目前建议隐藏以保持纯粹）。

### 2.4 提示词优化器 (Prompt Optimizer)

独立的文本处理流，从“图片处理”中发送图片或提示词过来。

- **顶部工具栏**：清空当前会话。右侧放置核心的 **“应用到工作区 (Apply to Workbench)”** 按钮。
- **左侧面板**：显示当前关联的参考图片（缩小为单列缩略图），供视觉参考。
- **中央工作区**：左右分栏或上下分屏的文本区。
  - 左/上：原始 Prompt 输入区。
  - 右/下：优化后的 Prompt 展示区（高亮显示新增或修改的关键词）。
- **右侧面板**：优化器配置区。包含 Refiner Model 选择、System Prompt 预设、Tag 过滤等控制项。

## 3. 多端响应式降级策略 (Responsive Degradation)

考虑到 Flutter 的特性，我们需要通过 `LayoutBuilder` 和屏幕宽度断点（Breakpoints）来实现流畅的跨端适配。

### 🖥️ Desktop (宽度 > 1024px)

- **布局状态**：完整展现“三栏一底”。左侧 App 全局导航在最左侧竖向排列。左右面板默认展开（允许用户手动折叠）。
- **交互逻辑**：主要依赖鼠标点击、悬停（Hover）显示操作按钮、拖拽调整面板宽度、右键菜单（辅助）。

### 📱 Pad / 平板横屏 (宽度 768px - 1024px)

- **布局状态**：保持 Desktop 类似布局，但空间更紧凑。
- **降级策略**：
  - 左侧目录树面板改为**默认折叠**的悬浮抽屉 (`Drawer`)。
  - 右侧属性面板保持固定，但宽度适当缩窄。
  - 取消强依赖 `Hover` 的交互（触屏无 Hover），所有图片卡片上的操作（对比、遮罩）必须**常驻显示**在卡片底栏，或者通过长按唤出上下文菜单。

### 📱 Mobile / 手机 (宽度 < 768px)

这是变化最大的视图，核心原则是**“单列垂直流”**和**“模态弹出”**。

- **App 全局导航**：从最左侧竖条降级为**底部导航栏 (`BottomNavigationBar`)**。
- **中央工作区**：占据除去顶栏和底栏的 100% 屏幕空间。
- **左侧面板 (目录树)**：彻底移除固定布局，降级为点击左上角汉堡菜单 (Hamburger Icon) 弹出的**全屏侧边抽屉 (`Drawer`)**。
- **右侧面板 (属性与操作)**：
  - 彻底移除固定布局。
  - 降级为**底部弹出面板 (`ModalBottomSheet`)**。例如，在“图片处理”中选好图片后，点击一个悬浮按钮 (FAB) 呼出底部面板，在里面选择模型和输入提示词。
- **子模块 Tab (顶部工具栏)**：改为顶部可水平滑动的 `TabBar`，或者集成到左上角的抽屉菜单中。
- **比较器/遮罩编辑器**：全屏模态显示 (`FullScreenDialog`)，覆盖底部的 App 导航栏，强制用户完成操作（保存/取消）后才能退出。
- **底部日志控制台**：移除常驻的物理形态。仅在出错时通过顶部的 `SnackBar` 或 `Toast` 弹出通知。提供一个独立的二级页面专门查看完整日志。
